## 핵심 제품 포지션

이 앱은 단순한 과제 관리 앱, 캘린더 앱, 일정 관리 앱이 아니다.

**"기록하는 앱이 아니라, 학생 대신 오늘의 학습 결정을 내리고 계획이 틀어졌을 때 다시 짜주는 앱"**

과제·시험·고정 일정·사용 가능한 시간을 종합해 학생 대신 "오늘 무엇을, 어떤 순서로, 얼마나 해야 하는지" 결정하고, 계획이 틀어졌을 때 자동으로 다시 계획을 짜주는 대학생용 학습 코치다.

---

## 모든 판단의 기준

새 기능이나 수정안을 제안하기 전에 반드시 다음을 판단한다:

1. 사용자가 오늘 무엇을 할지 고민하는 시간을 줄이는가?
2. 과제, 시험, 고정 일정, 가용 시간을 함께 고려하는가?
3. 단순 순위가 아니라 실제 실행 시간과 순서를 제안하는가?
4. 완료, 미완료, 일정 변경 시 계획을 자동으로 재조정하는가?
5. 추천 이유를 사용자가 이해할 수 있게 설명하는가?
6. 사용 기록이 쌓일수록 개인별 시간 예측이 정확해지는가?

위 기준을 강화하지 않는 기능은 우선순위를 낮추거나 제외한다.

---

## 핵심 기능 (중요도 순)

1. **실행 가능한 오늘 계획** — 단순 과제 순위가 아닌 실행 순서 + 시간 배정
2. **시험 복습 블록 생성** — 시험 날짜·범위·준비 기간 분석 → 오늘 복습 단위 생성
3. **과제 진행량 저장** — 오늘 배정 시간 + 실제 수행 시간 기록, 남은 시간 기준으로 재추천
4. **계획 자동 재조정** — 완료·일정 추가·가용 시간 변경 시 즉시 재계산
5. **현실적인 실패 대응** — 미완료 경고 대신 내일 이후로 현실적 재분산
6. **추천 근거 공개** — 마감일·난이도·예상 시간·시험 충돌·고정 일정 근거 표시
7. **개인화된 시간 예측** — 예상 vs 실제 수행 시간 비교, 개인별 소요시간 학습

---

## 개발 우선순위 로드맵

1. 시험을 오늘의 복습 블록으로 변환
2. 과제 진행량과 실제 수행 시간 저장
3. 못 끝낸 작업 자동 재배치
4. 사용자 행동 기반 예상 소요시간 보정
5. 학기 단위 워크스페이스

> 기존 기능의 심각한 버그, 보안 문제, 데이터 손실 가능성은 위 순서보다 먼저 처리한다.

---

## 피해야 할 방향

핵심 가치에 직접 기여하지 않는 한 다음은 확장하지 않는다:

- 범용 개인 일정 관리
- 시간표 앱 전체 대체
- 노션 같은 자유로운 문서 기능
- 목적 없는 일반 AI 채팅
- 과도한 통계와 그래프
- 캘린더 자체를 핵심 제품으로 확장
- 사용자가 직접 관리해야 할 설정·입력 항목 증가
- 기능 개수를 늘리기 위한 기능 추가

---

## 기능 제안·리뷰 시 응답 순서

1. 핵심 제품 방향과 맞는지 판단
2. 사용자의 결정 시간을 실제로 줄이는지 설명
3. 과제·시험·일정·가용 시간과 어떻게 연결되는지 설명
4. 구현 우선순위 명확히 제시
5. 범위가 커지거나 정체성이 흐려지면 분명하게 지적
6. 핵심 방향에 맞지 않는 기능은 억지로 긍정하지 말고 제외 권고

---

## 기존 코드 취급 원칙

- 기존 코드를 삭제하거나 처음부터 다시 구현하지 않는다
- 현재 코드와 구현된 기능을 먼저 읽고 이해한 뒤 수정한다
- 기존 기능이 방향과 맞지 않더라도 임의로 삭제하거나 대규모 리팩터링하지 않는다
- 먼저 문제점과 유지·축소·제거 중 적절한 선택을 설명한다
- 한 번에 전체 로드맵을 구현하지 말고, 핵심 가치에 가장 직접적으로 기여하는 최소 단위부터 작업한다

### F-22: 과제 이미지 파싱 (AI)

**설명**: 과제 공지 캡처 이미지를 업로드하면 AI가 과제 정보를 자동 추출한다.

**입력**

| 항목 | 제약 |
|---|---|
| 이미지 파일 | JPG / PNG / WEBP, 최대 5MB |

**처리 흐름**

1. 이미지 → Supabase Storage (assignment-image 버킷) 업로드
2. Signed URL 생성 (60초 유효)
3. Groq Vision API 호출 (meta-llama/llama-4-scout-17b-16e-instruct)
4. 응답 JSON 파싱 및 유효성 검증
5. 폼 필드 자동 채움

**AI 응답 구조**

```json
{
  "title": "과제 제목",
  "subject": "과목명",
  "dueDate": "YYYY-MM-DD",
  "difficulty": "easy|medium|hard",
  "estimatedHours": 3.0
}
```

**유효성 검증**

| 필드 | 검증 규칙 |
|---|---|
| difficulty | easy/medium/hard 외에는 "medium"으로 대체 |
| estimatedHours | 숫자 아니거나 0 이하 → 1.0, 0.5 단위 반올림 |
| dueDate | YYYY-MM-DD 형식 아니거나 Invalid Date → 오늘+7일 |

**에러 처리**

| 케이스 | 사용자 메시지 |
|---|---|
| 파일 형식 오류 | "JPG, PNG, WEBP 이미지만 지원합니다." |
| 파일 크기 초과 | "파일 크기는 5MB 이하여야 합니다." |
| 업로드 실패 | "이미지 업로드 실패: {에러 메시지}" |
| AI 응답 파싱 실패 | "AI 응답 파싱 실패" |
| 정보 추출 불가 | "과제 정보를 추출하지 못했습니다." |

### F-40 + F-61: 오늘 플래너 (로컬 스코어링 + AI 코멘트)

**구조**: 두 단계로 분리되어 있음. **UI는 하나의 화면에 통합.**

#### 1단계 — 로컬 우선순위 스코어링 (`priority.ts: recommend()`)

우선순위 계산은 AI가 아닌 **로컬 알고리즘**이 수행. Groq API 호출 없이 즉시 결과 반환.

**스코어 계산식**

```
score = urgencyScore(daysLeft) × 0.60
      + DIFFICULTY_SCORE[difficulty] × 0.25
      + hoursScore(estimatedHours) × 0.15
```

| 요소 | 가중치 | 기준 |
|---|---|---|
| 긴급도 (마감 잔여일) | 60% | D-0: 100점, D-1: 92점, D-2: 80점, D-4: 60점, D-7: 35점 |
| 난이도 | 25% | hard: 100점, medium: 60점, easy: 20점 |
| 소요시간 | 15% | 짧을수록 높음 (≤1h: 100점, ≤2h: 80점) |

스코어 정렬 후 탐욕 알고리즘으로 가용 시간 배정 (시간 초과 시 부분 배정).

**입력**: 과제 목록 + 가용 시간 (사용자 입력)  
**출력**: `ScoredAssignment[]` — 순위, 배정 시간, 추천 이유 태그 포함

#### 2단계 — AI 코멘트 (`ai.ts: getAIComment()`)

1단계 결과를 받아 Groq API에 **코멘트 생성만** 요청. 순위 변경 없음.

**모델**: llama-3.3-70b-versatile (Groq)  
**입력**: 이미 계산된 추천 목록 + 가용 시간  
**출력**: 한국어 학습 코멘트 (3~5문장)

### F-60: AI 우선순위 엔진 v2 (예정 — W4)

**설명**: 시험 일정 충돌을 반영한 정밀 우선순위 스코어링

**스코어링 요소**

| 요소 | 가중치 방향 |
|---|---|
| 마감 잔여일 | 짧을수록 높음 |
| 난이도 | hard > medium > easy |
| 예상 소요시간 | 길수록 조기 착수 유도 |
| 시험 충돌 | 시험 D-3 이내에 과제 마감 겹치면 우선순위 상향 |

**테스트 시나리오**

| 시나리오 | 기대 결과 |
|---|---|
| 마감 D-1 쉬운 과제 vs 마감 D-5 어려운 과제 | D-1 과제 우선 |
| 시험 전날 과제 마감 | 시험 대비 감안한 복합 추천 |
| 모든 과제 마감 동일일 | 난이도 높은 것 우선 |
| 이미 in_progress 과제 vs 새 과제 | 진행 중인 것 우선 |

## 5. 사용자 스토리 (User Stories)

```
US-01: 과제 공지 캡처
  As a 대학생
  I want to 과제 공지 캡처를 업로드하면 AI가 내용을 자동 입력해주길
  So that 매번 직접 타이핑하지 않아도 된다

US-02: 오늘 할 일 판단
  As a 대학생
  I want to 앱이 "오늘 뭘 먼저 해야 하는지"를 알려주길
  So that 내가 직접 우선순위를 고민하는 시간을 줄일 수 있다

US-03: 시험 충돌 알림
  As a 대학생
  I want to 과제 마감일과 시험 날짜가 겹칠 때 AI가 경고해주길
  So that 마감 직전에 당황하지 않는다

US-04: 과제 완료 추적
  As a 대학생
  I want to 완료한 과제를 체크하고 진행률을 볼 수 있길
  So that 뭘 했고 뭐가 남았는지 한눈에 파악할 수 있다
```

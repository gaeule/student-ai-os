## 4. 데이터 흐름

### 4.1 과제 입력 (이미지 파싱)

```
사용자 이미지 선택
  → 브라우저: File → URL.createObjectURL (미리보기)
  → [AI 자동 입력] 클릭
  → FormData(image) → Server Action: parseAssignmentImage()
    → Supabase Storage 업로드 (assignment-image 버킷)
    → Signed URL 생성 (60초 유효)
    → Groq Vision API 호출 (llama-4-scout)
    → JSON 파싱 + 유효성 검증
    → { data: ParsedAssignment } 반환
  → 클라이언트: 폼 필드 자동 채움 (setValue)
  → 사용자 확인/수정 후 [과제 등록] 클릭
  → Server Action: createAssignment()
    → Supabase DB INSERT
    → revalidatePath("/assignments"), revalidatePath("/dashboard")
```

### 4.2 로컬 스코어링 + AI 코멘트

우선순위 **계산**과 **코멘트 생성**은 완전히 분리되어 있음.

```
[추천받기] 버튼 클릭 (클라이언트)
  → 1단계: priority.ts: recommend(assignments, hours) 즉시 실행 (API 호출 없음)
      → 마감일/난이도/소요시간 가중 스코어 계산
      → 탐욕 알고리즘으로 시간 배정
      → ScoredAssignment[] 즉시 화면에 표시
  → 2단계: Server Action: getAIComment(recommendations, hours) 비동기 호출
      → 이미 계산된 추천 목록을 Groq API (llama-3.3-70b)에 전달
      → 프롬프트: "이 추천 결과에 대한 학습 코멘트를 작성해줘"
      → 텍스트 코멘트만 반환 (순위 변경 없음)
      → AICommentBox에 로딩 → 완료 순으로 표시
```

> Groq가 우선순위를 결정하지 않음. 순서는 priority.ts 로컬 알고리즘이 결정.

### 4.3 인증 흐름

```
미인증 접근 → middleware.ts → /login 리다이렉트
로그인 성공 → Supabase Auth → 쿠키 저장
  → /auth/callback → 세션 교환 → /dashboard
Google OAuth → Supabase OAuth → 콜백 처리
```

## 5. 보안 설계

### 5.1 Row Level Security (RLS)

모든 테이블에 RLS 활성화. 각 테이블의 CRUD는 `user_id = auth.uid()` 조건으로 제한.

```sql
-- 예시: assignments 테이블 SELECT 정책
CREATE POLICY "users can view own assignments"
ON assignments FOR SELECT
USING (user_id = auth.uid());
```

### 5.2 Server Action 보안

모든 Server Action에서 `supabase.auth.getUser()` 호출 후 user 확인:

```typescript
const { data: { user } } = await supabase.auth.getUser();
if (!user) return { error: "로그인이 필요합니다" };
```

### 5.3 이미지 업로드 보안

- MIME 타입 검증: `image/jpeg`, `image/png`, `image/webp`만 허용
- 파일 크기 제한: 5MB 이하
- 경로: `{user_id}/{uuid}.{ext}` — 사용자별 격리
- Private 버킷 + Signed URL (60초 만료)

### 5.4 서버 설정

```typescript
// next.config.ts
experimental: {
  serverActions: { bodySizeLimit: "5mb" }
}
```

## 6. 성능 고려사항

### 6.1 Turbopack

개발 서버에 Turbopack 적용 (`--turbopack` 플래그). 메모리 사용량 대폭 감소 (webpack 대비).

### 6.2 Server Actions 캐싱

- `revalidatePath()`: 과제/과목 변경 시 관련 페이지 캐시 무효화
- AI 코멘트: 현재 매 요청마다 호출 (향후 캐싱 검토)

### 6.3 이미지 URL 메모리 관리

```typescript
// URL.createObjectURL 메모리 누수 방지
useEffect(() => {
  return () => { if (imagePreview) URL.revokeObjectURL(imagePreview); };
}, [imagePreview]);
```

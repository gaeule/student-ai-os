## 1. 전체 화면 구조 (Site Map)

```
/                           → 랜딩 페이지 (미로그인 시 표시)
/login                      → 로그인
/signup                     → 회원가입
/auth/callback              → OAuth 콜백 처리

/(app)                      → 인증된 사용자 전용
  /dashboard                → 대시보드 (메인)
  /assignments              → 과제 관리
  /subjects                 → 과목 관리
  /today                    → 오늘 할 일 플래너
  /exams                    → 시험 일정 (예정)
```

## 3. 화면별 상세 명세

---

### 3.1 랜딩 페이지 (`/`)

**목적**: 미로그인 사용자 전환 (로그인/회원가입 유도)

| 요소 | 내용 |
|---|---|
| 헤드라인 | "오늘 뭘 먼저 해야 할지 AI가 알려드릴게요" |
| CTA 버튼 | "시작하기" → /signup |
| 보조 링크 | "이미 계정이 있나요?" → /login |

---

### 3.2 로그인 / 회원가입 (`/login`, `/signup`)

**현재 구현**: 이메일 + Google OAuth

| 요소 | 로그인 | 회원가입 |
|---|---|---|
| 입력 필드 | 이메일, 비밀번호 | 이메일, 비밀번호, 비밀번호 확인 |
| 소셜 버튼 | Google 로그인 | Google 가입 |
| 에러 표시 | 인라인 에러 메시지 | 인라인 에러 메시지 |

---

### 3.3 대시보드 (`/dashboard`)

**목적**: 전체 현황 한눈에 파악

#### 영역 구성

```
┌─────────────────────────────────────────┐
│  통계 카드 (4개)                         │
│  [이번 주 과제] [마감 임박] [진행 중] [완료율] │
├─────────────────────────────────────────┤
│  오늘 할 일 미리보기                      │
│  (우선순위 TOP 3 과제)                   │
└─────────────────────────────────────────┘
```

#### 통계 카드 (StatsGrid.tsx 실제 구현 기준)

| 카드 | 표시 내용 | 아이콘 | 강조 조건 |
|---|---|---|---|
| 이번 주 과제 | 이번 주(월~일) 마감 과제 수 | FileText | - |
| 마감 임박 | D-3 이내 미완료 과제 수 | AlertTriangle | 1개 이상이면 빨간색 |
| 진행 중 | 전체 과제 중 미완료 수 | TrendingUp | - |
| 완료율 | done / 전체 % + 프로그레스 바 | CheckCircle2 | - |

> **시험 예정 카드 없음**: 코드에 없음. exams 기능 구현 후 추가 예정.

#### 오늘 할 일 미리보기

- 우선순위 상위 3개 과제 카드
- 각 카드: 과제명, 과목명, 마감일, 난이도 뱃지
- "오늘 플래너 열기" 버튼 → /today

### 3.4 과제 관리 (`/assignments`)

**목적**: 과제 전체 CRUD

#### 화면 구성

```
┌─────────────────────────────────────────┐
│  헤더: "과제 관리" + [+ 과제 추가] 버튼  │
├──────────────────┬──────────────────────┤
│  과제 입력 폼    │  과제 목록             │
│  (사이드 패널)   │  (메인 영역)           │
└──────────────────┴──────────────────────┘
```

#### 과제 입력 폼

| 필드 | 타입 | 필수 |
|---|---|---|
| 이미지 업로드 | 파일 (JPG/PNG/WEBP, 5MB) | 선택 |
| AI 자동 입력 버튼 | 트리거 버튼 | - |
| 과제명 | 텍스트 | 필수 |
| 과목 | 셀렉트 (subjects 목록) | 필수 |
| 마감일 | 달력 팝오버 | 필수 |
| 난이도 | 토글 버튼 (상/중/하) | 필수 |
| 예상 소요시간 | 숫자 입력 (시간 단위) | 필수 |

#### 과제 목록 아이템

| 요소 | 내용 |
|---|---|
| 과제명 | 텍스트 |
| 과목명 | 서브텍스트 (회색) |
| 마감일 | "D-N" 형식, 임박 시 빨간색 |
| 난이도 뱃지 | 상(빨강) / 중(노랑) / 하(초록) |
| 소요시간 | "N시간" |
| 상태 | todo → in_progress → done 토글 |
| 수정 버튼 | 연필 아이콘 → 인라인 편집 모드 |
| 삭제 버튼 | 휴지통 아이콘 |

#### 편집 모드 (인라인)

- 선택한 과제 아래 폼 펼침
- 이미지 업로드 섹션 숨김 (편집 시 불필요)
- 저장/취소 버튼

### 3.5 과목 관리 (`/subjects`)

**목적**: 과목 CRUD

#### 화면 구성

```
┌─────────────────────────────────────────┐
│  헤더: "과목 관리" + [+ 과목 추가] 버튼  │

│  신규 등록 폼 (토글)                     │
│  ┌─────────────────────────────────┐   │

├─────────────────────────────────────────┤
│  과목 목록                              │

```

#### 과목 필드

| 필드 | 타입 | 필수 |
|---|---|---|
| 과목명 | 텍스트 | 필수 |
| 교수명 | 텍스트 | 선택 |
| 학기 | 텍스트 (예: 2026-1) | 선택 |

### 3.6 오늘 할 일 플래너 (`/today`)

**목적**: AI 기반 오늘 시간 배분 추천

#### 화면 구성

```
┌─────────────────────────────────────────┐
│  헤더: "오늘 할 일"  [날짜]             │
├─────────────────────────────────────────┤
│  가용 시간 입력                          │
│  [오늘 공부할 수 있는 시간: ___시간]      │
│  [AI 추천 받기] 버튼                     │
├─────────────────────────────────────────┤
│  AI 추천 결과 카드                       │
│  ┌───────────────────────────────┐      │
│  │ 1순위: DB 과제 — 2시간        │      │
│  │   이유: 마감 D-2, 난이도 상    │      │
│  ├───────────────────────────────┤      │
│  │ 2순위: 영어 에세이 — 1시간     │      │
│  └───────────────────────────────┘      │
├─────────────────────────────────────────┤
│  AI 코멘트 (학습 코치)                   │
│  "마감이 2개 겹치는 날이에요. 오늘..."   │
└─────────────────────────────────────────┘
```

#### AI 추천 카드 항목

| 요소 | 내용 |
|---|---|
| 순위 | 1순위, 2순위... |
| 과제명 | 텍스트 |
| 추천 시간 | N시간 배분 |
| 이유 | 마감일, 난이도, 시험 충돌 근거 |

## 4. 상태 흐름 (State Flow)

### 과제 상태

```
todo → in_progress → done
 ↑                    ↓
 └──── (재오픈) ───────┘
```

### 이미지 파싱 흐름

```
이미지 선택 → 미리보기 표시 → [AI로 자동 입력] 클릭
  → 로딩 (분석 중...) → 필드 자동 채움 → 사용자 확인/수정
```

## 5. 빈 상태 (Empty State) 처리

| 화면 | 빈 상태 메시지 |
|---|---|
| 과제 목록 | "등록된 과제가 없습니다. 첫 과제를 등록해보세요." |
| 과목 목록 | "등록된 과목이 없습니다. 과목 추가 버튼을 눌러 등록해보세요." |
| 오늘 플래너 | "등록된 과제가 없습니다. 먼저 과제를 추가해주세요." |
| 대시보드 미리보기 | "오늘 추천할 과제가 없습니다." |
